# 作业报告图片自行截图要求

本文件用于替换《大模型技术在系统中的应用说明报告.docx》中原来参考 PDF 生成的图片。提交时不要使用附件 PDF 页面截图，也不要裁剪 PDF 中的图片；每张图都应来自你自己在本机 Docker、Dify、Ollama 或浏览器中的实际操作界面。

## 一、通用截图要求

1. 截图前建议把浏览器窗口最大化，缩放比例保持 90% 或 100%，分辨率尽量使用 1920×1080 或接近 16:9 的窗口。
2. 截图时只保留关键界面，不需要截整个桌面；可以使用 Windows 的 `Win + Shift + S` 框选区域。
3. 图片中应能看清页面标题、关键按钮、模型名称、应用名称、知识库名称、节点名称或问答结果。
4. 涉及账号、邮箱、密钥、数据库密码、内网地址等敏感信息时，请先打码或避免截入画面。
5. 建议把所有截图保存到项目目录下新建文件夹 `my_screenshots`，文件名按本文要求命名，例如 `fig01_dify_deploy.png`。
6. 如果某一张报告图需要两个界面共同说明，可以截两张原始图，再在 Word 中分别插入为两个小图，或用截图工具/画图软件上下拼成一张图。不要使用 PDF 中的示例图拼接。

## 二、截图前准备

1. 启动 Docker Desktop，确认 Dify 相关容器已经运行。
2. 启动 Ollama，并确认本地模型可用。可在命令行执行 `ollama list`，看到 `deepseek-r1:1.5b` 或你实际使用的模型名称。
3. 打开浏览器访问本地 Dify。首次初始化一般访问 `http://localhost/install`；已经初始化过则访问 `http://localhost/signin` 或 `http://localhost/apps`。
4. 如果你的 Dify 页面文字或菜单名称与本文略有不同，以实际界面为准，但截图内容要覆盖相同功能点。

## 三、逐图截图清单

### 图 1 本地部署 Dify 并初始化管理员账号

建议文件名：`fig01_dify_deploy.png`

需要展示的内容：Dify 本地服务已经启动，并且可以进入初始化或登录页面。

截图产出方式：

1. 打开 Docker Desktop。
2. 点击左侧 `Containers`。
3. 找到 Dify 项目相关容器，通常能看到 `api`、`web`、`worker`、`db`、`redis`、`nginx` 等容器处于 Running 状态。
4. 截图 Docker Desktop 中 Dify 容器运行列表，保存为 `fig01a_docker_running.png`。
5. 打开浏览器访问 `http://localhost/install`。如果已经完成初始化，访问 `http://localhost/signin`，展示 Dify 登录页面即可。
6. 截图 Dify 初始化管理员账号页面或登录页面，保存为 `fig01b_dify_install_or_login.png`。
7. 最终在报告中可以把这两张图上下排列，或合成为 `fig01_dify_deploy.png`。

截图范围要求：Docker 图要截到容器名称和运行状态；Dify 图要截到地址栏中的 localhost 和页面主体，不需要截整个桌面。

### 图 2 在 Dify 中关联本地 DeepSeek 模型

建议文件名：`fig02_model_provider.png`

需要展示的内容：Dify 已经配置本地 Ollama/DeepSeek 模型，系统模型或供应商页面能看到模型名称。

截图产出方式：

1. 登录 Dify。
2. 点击右上角头像或用户名入口。
3. 进入 `设置`。
4. 点击 `模型供应商`。
5. 找到 `Ollama` 或本地模型供应商配置项。
6. 如果页面有 `系统模型设置`，点击进入，选择或查看 `deepseek-r1:1.5b`；如果你实际使用其他模型，显示你的实际模型名称。
7. 截图模型供应商配置页或系统模型设置弹窗，保存为 `fig02_model_provider.png`。

截图范围要求：必须能看到 `Ollama`、`deepseek-r1:1.5b` 或实际模型名，以及 Dify 设置页面上下文。

### 图 3 创建聊天助手并配置模型参数

建议文件名：`fig03_chat_app_config.png`

需要展示的内容：在 Dify 中创建聊天助手应用，并配置大模型参数。

截图产出方式：

1. 在 Dify 左侧或顶部进入 `工作室`。
2. 点击 `创建空白应用`。
3. 选择 `聊天助手`。
4. 输入应用名称，例如 `智能聊天机器人`；描述可填写 `用于测试本地大模型问答能力`。
5. 点击 `创建`。
6. 进入应用编排或配置页面后，在右上角或模型设置区域选择本地模型，例如 `deepseek-r1:1.5b`。
7. 展开参数设置区域，显示温度、Top P、上下文长度等参数。
8. 截图应用配置页面，保存为 `fig03_chat_app_config.png`。

截图范围要求：图中要能看到应用名称、聊天助手页面，以及模型选择或参数配置区域。若创建应用弹窗和模型配置不在同一页，可分别保存为 `fig03a_create_chat_app.png` 与 `fig03b_model_params.png`。

### 图 4 上传资料并完成知识库向量化配置

建议文件名：`fig04_knowledge_indexing.png`

需要展示的内容：创建知识库、上传自己准备的资料，并完成解析、分段、向量化或索引配置。

截图产出方式：

1. 在 Dify 中进入 `知识库`。
2. 点击 `创建知识库`。
3. 选择 `导入已有文本` 或 `上传文件`。
4. 建议使用自己整理的课程资料文本，例如新建一个 `大模型技术知识库资料.txt`，内容可写 Dify、Ollama、RAG、MCP 的简要说明。也可以上传你自己准备的 Word/PDF 资料，但不要把附件 PDF 的页面截图作为报告图片。
5. 上传文件后点击 `下一步`。
6. 在索引配置页面选择 Embedding 模型、检索方式、分段规则等参数。
7. 点击保存或开始处理，等待解析和向量化完成。
8. 截图上传文件页面或索引配置页面，保存为 `fig04a_upload_material.png`。
9. 截图处理完成后的文档列表、分段预览或知识库详情页面，保存为 `fig04b_index_finished.png`。
10. 最终在报告中可以把两张图上下排列，或合成为 `fig04_knowledge_indexing.png`。

截图范围要求：要能看到知识库名称、上传的文件名、Embedding/索引配置或处理完成状态。

### 图 5 将知识库添加到聊天应用并进行问答测试

建议文件名：`fig05_rag_chat_test.png`

需要展示的内容：聊天助手已经接入知识库，并能根据知识库内容回答问题。

截图产出方式：

1. 回到 `工作室`。
2. 打开前面创建的 `智能聊天机器人` 应用。
3. 在应用编排或上下文设置区域点击 `添加知识库`。
4. 选择图 4 中创建的知识库。
5. 保存应用设置。
6. 进入右侧预览窗口或调试聊天窗口。
7. 输入与知识库内容相关的问题，例如 `Dify 知识库应用的工作流程是什么？` 或 `RAG 在系统中起什么作用？`
8. 等待模型回复，最好展开或显示引用片段、检索结果或上下文来源。
9. 截图问答结果页面，保存为 `fig05_rag_chat_test.png`。

截图范围要求：图中应同时包含用户问题、模型回答，以及能体现已添加知识库的区域。如果引用来源可以展开，优先展开后再截。

### 图 6 SQL 查询工作流的节点编排

建议文件名：`fig06_sql_workflow_nodes.png`

需要展示的内容：使用 Dify 工作流把自然语言输入、LLM 理解、参数提取和 SQL 查询节点串联起来。

截图产出方式：

1. 在 Dify 进入 `工作室`。
2. 点击 `创建空白应用`。
3. 选择 `工作流` 或 `Chatflow`。不同版本名称可能不同，选择能拖拽节点编排的应用类型即可。
4. 创建应用名称，例如 `SQL 查询器`。
5. 在画布中保留 `开始` 节点。
6. 添加 `LLM` 节点，用于根据用户问题生成 SQL。
7. 添加 `参数提取器` 节点，用于提取 SQL 字段。
8. 添加 `SQL 查询`、`数据库查询` 或类似数据库工具节点。如果本地没有数据库连接，也可以先把节点配置页打开到连接信息处，但截图中要体现该节点存在。
9. 将节点按 `开始 -> LLM -> 参数提取器 -> SQL 查询` 顺序连接。
10. 调整画布缩放，让所有节点都在同一屏内。
11. 截图工作流画布，保存为 `fig06_sql_workflow_nodes.png`。

截图范围要求：必须能看清节点名称和连接线。若 SQL 节点无法配置成功，截图仍需展示工作流设计结构，并在报告说明中写为“工作流设计与配置界面”。

### 图 7 使用参数提取器提取 SQL 语句

建议文件名：`fig07_parameter_extractor.png`

需要展示的内容：参数提取器节点中配置了 SQL 提取参数和提取指令。

截图产出方式：

1. 打开图 6 的 `SQL 查询器` 工作流。
2. 点击画布中的 `参数提取器` 节点。
3. 在右侧配置面板中新增或查看提取参数。
4. 参数名称建议设置为 `sql` 或 `sql_list`。
5. 参数说明可填写 `SQL query statement, each item contains one executable SQL statement`。
6. 参数类型建议选择数组类型；如果页面不支持数组，则选择字符串并在指令中要求只输出 SQL。
7. 在提取指令中写明从 LLM 返回内容中提取 SQL 查询语句，例如：`请从模型输出中提取可执行的 SQL 查询语句，只返回 SQL 数组，不要返回解释文本。`
8. 截图参数提取器配置面板，保存为 `fig07_parameter_extractor.png`。

截图范围要求：要能看到节点名称、参数名 `sql` 或 `sql_list`、参数类型/说明、提取指令中的关键文字。

### 图 8 MCP 用于连接外部工具与数据源

建议文件名：`fig08_mcp_agent_tools.png`

需要展示的内容：Dify 的 Agent 或插件/工具页面中展示 MCP、外部工具、工具调用或 Agent 构建能力。

截图产出方式：

1. 优先方案：在 Dify 中进入 `插件`、`工具` 或 `Marketplace`。
2. 搜索 `MCP`。
3. 如果能看到 MCP 插件、MCP SSE、MCP 工具或相关工具项，打开详情页或配置页。
4. 截图插件详情或工具配置页面，保存为 `fig08_mcp_agent_tools.png`。
5. 如果你的 Dify 版本没有 MCP 插件入口，则进入 `工作室`，点击 `创建空白应用`。
6. 选择 `Agent` 或带工具调用能力的应用类型。
7. 进入 Agent 编排页面后，打开 `工具` 或 `Tools` 配置区域，展示模型可以接入外部工具。
8. 截图 Agent 工具配置页面，仍保存为 `fig08_mcp_agent_tools.png`。

截图范围要求：优先截到 `MCP` 字样；如果页面没有 MCP 字样，则至少要截到 Agent 的工具配置区域，并在报告文字中写为“通过 Agent/工具调用机制为后续接入 MCP 提供扩展入口”。

## 五、截图后替换报告图片的原则

1. Word 报告中的图题可以继续沿用原来的 `图 1` 到 `图 8`。
2. 替换图片时保持图片居中，宽度尽量一致，避免一张很大一张很小。
3. 如果某张图与原图内容略有差异，以你真实操作为准，报告文字中把模型名、应用名、知识库名改成截图里的实际名称。
4. 截图必须体现“系统中如何使用大模型技术”：模型接入、聊天问答、知识库检索、工作流编排、参数提取、工具/Agent 扩展这几类内容至少要覆盖。
